Safety & workspace scope

Cleanup that cannot escape its workspace

A maintenance command could quietly reason about the wrong repository. This change makes the invoking workspace the hard boundary for every cleanup decision.

The problem was not deletion itself. It was that the command could establish its scope from maintainer state instead of the place where the user invoked it.
